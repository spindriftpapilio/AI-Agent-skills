# 工作区信任与审批状态清单

测试时建议至少记录这几个状态：

1. 未打开项目
2. 已打开但未信任
3. 已信任但未批准任何高风险动作
4. 已批准部分工具或部分MCP
5. 修改配置后再次使用
6. 重开项目后再次使用

每个状态下都问这几个问题：

- 会读哪些文件
- 会起哪些组件
- 会发哪些请求
- 会记住哪些批准
- 会阻止哪些敏感写入

## 最小核对项

|检查项|要点|
|---|---|
|信任提示出现时机|是在任何项目级配置生效之前，还是之后|
|批准缓存粒度|按目录、按服务、按工具、按命令、按文件，还是按内容|
|配置变更后行为|旧批准是否会覆盖新内容|
|重启后的行为|批准是否跨会话复用|
|dotfile保护|编辑与创建是否同样受控|
