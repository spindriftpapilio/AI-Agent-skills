# AI Agent测试skill

这是一组面向AI Agent安全测试评估的skills。

重点覆盖这些方向：

- 攻击面侦察
- 提示词注入与持久化
- MCP配置投毒
- 终端过滤绕过
- 数据外传
- 代码执行入口
- 工作区信任与审批
- Hook与子Agent
- 浏览器与CI流程
- skill与配置供应链
- 多阶段攻击链编排
- 有源码场景下的定向审计

---

## 适用范围

这套skill适合拿来测各类AI Agent与Agent型开发工具，例如：

- 编码Agent
- CLI Agent
- IDE内置Agent
- 带工具调用能力的Agent
- 支持MCP、Hook、规则文件、项目级配置的Agent产品
- 支持Computer Use、浏览器代操作、CI集成、消息通道集成的Agent产品

如果目标具备下面这些能力，这套内容会更有用：

- 会读工作区文件
- 会写配置、脚本或规则文件
- 会调用终端、外部工具或本地程序
- 会访问网络或加载远程资源
- 会接触浏览器登录态、CI令牌、消息通道或自动预览机制
- 有审批、信任、沙箱或权限缓存模型

## 仓库内容

当前仓库包含12个skill目录：

| 目录 | 说明 |
| --- | --- |
| `AI-IDE攻击面侦察` | 先梳理目标会读什么、写什么、执行什么、联网什么 |
| `提示词注入攻击链测试` | 测提示词注入能否影响真实行为、改写配置、形成持久化 |
| `MCP配置投毒测试` | 测不可信MCP定义是否会被发现、加载、调用 |
| `终端过滤绕过测试` | 测命令过滤、参数校验、Shell包装是否能被绕过 |
| `AI-IDE数据外传测试` | 测图片、WebView、URL拉取、DNS等出站通道 |
| `AI-IDE代码执行面测试` | 测Hook、设置、工具定义、路径劫持等执行入口 |
| `AI-IDE源码安全审计` | 有源码时，围绕配置、权限、执行边界做定向审计 |
| `AI-IDE攻击链编排` | 把单点能力串成完整利用链，判断真实影响 |
| `Agent工作区信任与审批测试` | 测信任提示、批准缓存、敏感文件保护、预信任执行 |
| `Agent Hooks与子Agent安全测试` | 测Hook、子Agent、任务分派与权限继承 |
| `Agent浏览器与CI流程安全测试` | 测浏览器Agent、Computer Use、CI提示词注入、截图注入与消息通道外传 |
| `Agent skill与配置供应链安全` | 测skills、规则文件、MCP配置与marketplace链路 |

说明：目录名里有一部分保留了历史命名，比如`AI-IDE`。实际使用时，可以把它们当成更广义的AI Agent测试主题来用。

## 推荐使用顺序

面对一个新Agent目标时，建议按这个顺序推进：

1. `AI-IDE攻击面侦察`
2. `提示词注入攻击链测试`
3. `MCP配置投毒测试`
4. `Agent工作区信任与审批测试`
5. `终端过滤绕过测试`
6. `AI-IDE数据外传测试`
7. `AI-IDE代码执行面测试`
8. `Agent Hooks与子Agent安全测试`
9. `Agent浏览器与CI流程安全测试`
10. `Agent skill与配置供应链安全`
11. `AI-IDE源码安全审计`（有源码时）
12. `AI-IDE攻击链编排`

可以按这个思路理解：

- 先弄清攻击面
- 再验证单点能力
- 再看这些能力能不能接起来
- 最后判断影响、触发条件和复现成本

## 方法重点

这套skill更看重真实运行边界，不只看表面功能说明。

重点通常包括：

- 哪些输入会进入Agent上下文
- 哪些配置会自动生效
- 哪些文件会被Agent代写
- 哪些工具能直接执行
- 哪些出站通道默认可用
- 哪些批准会被缓存或错误复用
- 哪些低风险原语能进一步升级
- 哪些浏览器、CI、消息通道能力会把风险放大

使用时建议遵循这些原则：

- 先看运行时行为，再看源码
- 先走通一条最小链路，再横向扩展
- 先确认触发条件，再谈风险放大
- 明确区分零交互、单次交互、需审批、已信任后触发

## 使用建议

- 这套仓库偏测试方法与研究资料，不是自动化扫描器。
- skill中的路径、配置名、样例格式需要按目标产品调整。
- 同一类Agent在不同版本、不同平台上的行为可能差很多，结论应以实测为准。
- 对浏览器Agent、CI Agent、消息通道型Agent，建议单独记录登录态、令牌范围、预览行为和出站能力。
