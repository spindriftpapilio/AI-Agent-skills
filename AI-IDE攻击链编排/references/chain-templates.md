# 攻击链模板

用于把已经确认的漏洞原语拼成端到端利用链。每个模板都包含：

- 前提条件
- 布置方式
- 触发动作
- 预期结果
- 分级说明

# 模板1：经典链（PI -> settings.json -> RCE）

**适用前提：**

- [ ]已确认提示词注入成立
- [ ]已确认Agent可以改写工作区设置文件
- [ ]已确认IDE会从工作区设置读取可执行路径

**基本流程：**
1. 在README、注释或代码中放入提示词注入内容
2. 诱导Agent写入工作区设置，例如`.vscode/settings.json`
3. 把某个解释器、校验器或工具路径改到恶意程序
4. 等待IDE自动触发该工具

**预期结果：**
- 打开项目、切换文件、启动扩展或运行检查时执行恶意程序

**常用落点：**
- `php.validate.executablePath`
- 语言服务、格式化器、lint工具路径
- task、Shell、terminal相关设置

# 模板2：PI ->配置投毒 ->自动加载

**适用前提：**

- [ ]Agent能写工作区配置
- [ ]IDE会自动加载该配置
- [ ]新配置能引入命令执行、工具调用或外传

**基本流程：**
1. 通过提示词注入让Agent改写MCP / 规则文件 / Hook / 设置
2. 关闭并重开项目，或等待热加载
3. 观察是否出现自动执行、自动联网或新工具可用

**预期结果：**
- 一次正常交互变成跨会话持久化影响

# 模板3：PI ->终端过滤绕过 ->执行

**适用前提：**

- [ ]模型可引导到终端命令
- [ ]终端过滤存在绕过点

**基本流程：**
1. 注入内容诱导Agent运行看起来合理的命令
2. 在允许命令中塞危险参数、Shell展开或结构绕过
3. 让实际执行的命令超出审批或过滤预期

**预期结果：**
- 从Agent能跑受限命令升级为Agent能跑任意或敏感命令

# 模板4：PI ->读敏感数据 ->外传

**适用前提：**

- [ ]Agent会读取工作区或环境中的敏感内容
- [ ]已确认存在稳定的出站通道

**基本流程：**
1. 诱导Agent读取敏感文件、配置或token
2. 借图片、URL、DNS、WebView、工具调用等方式带出

**预期结果：**
- 数据离开本地或沙箱边界

# 模板5：先审批，后替换（TOCTOU）

**适用前提：**

- [ ]产品存在审批缓存
- [ ]审批不绑定内容哈希

**基本流程：**
1. 先让用户批准一个看似无害的配置或命令
2. 再通过拉取更新、二次写入或路径替换改掉内容
3. 观察系统是否沿用旧审批

**预期结果：**
- 后续恶意内容在已批准状态下静默运行

# 编写攻击链时宜记录的内容

- 每一跳的输入是什么
- 哪一跳是状态边界变化点
- 是否需要用户额外动作
- 终点影响是读、写、执行还是持久化
- 从干净状态能否稳定复现
