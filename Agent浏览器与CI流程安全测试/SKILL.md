# Agent浏览器与CI流程安全测试

本skill聚焦一类很容易被低估、但实战价值很高的场景：Agent不只读本地文件，还会读网页、截图、PR描述、Issue内容、流水线日志、聊天消息和预览结果，并在这些上下文里继续做工具调用或执行动作。

一旦目标同时具备下面三类能力，风险通常会明显上升：

- 会读取不可信外部内容
- 会接触敏感数据或高权限令牌
- 会调用浏览器、终端、CI工具、消息通道或网络请求

# 什么时候用

- 目标支持浏览器Agent、Computer Use、网页总结、网页代操作
- 目标会在CI/CD中运行Agent，例如处理Issue、PR、提交信息、代码审查结果
- 目标支持通过Slack、Telegram、邮件、工单系统等渠道与Agent交互
- 目标会读取截图、OCR结果、PDF、网页注释、评论区等不可信内容
- 你已经确认存在读上下文、发请求、执行命令或写配置能力，想判断是否能形成更贴近真实使用场景的攻击链

# 核心问题

1. 不可信外部内容会通过什么路径进入Agent上下文
2. Agent读取这些内容后，能调用哪些高风险工具
3. 浏览器会不会带着已登录状态跨站访问敏感页面
4. CI里的Agent是否能接触高权限令牌、仓库密钥、云凭证或制品发布权限
5. 聊天通道、链接预览、截图解析是否会在用户几乎无感的情况下完成出站
6. 批准、沙箱、工作区信任是否真的能挡住这类链路

# 交互层级

|层级|典型场景|
|---|---|
|Tier 1|打开网页、打开仓库、打开消息后自动读入恶意内容|
|Tier 2|用户点一次总结、分析、审查、回复或运行流水线|
|Tier 3|用户需要批准一次工具调用、浏览器动作或命令执行|
|Tier 4|用户已信任目标，且还要进行额外操作，如登录高敏站点或合并PR|

# 推荐流程

1. **先画内容进入路径。**
   列清楚网页、截图、Issue、PR、日志、聊天消息、链接预览、OCR文本分别从哪里进上下文。

2. **再画权限图。**
   重点记录：
   - 浏览器是否共享主浏览器登录态
   - 是否可访问邮箱、代码托管、工单、云控制台
   - CI中是否暴露令牌、仓库写权限、包发布权限
   - 消息通道是否默认启用链接预览

3. **确认真实执行点。**
   看Agent最终能落到哪里：
   - 浏览器动作
   - 命令执行
   - MCP工具
   - 外部HTTP / DNS请求
   - 发消息、回帖、写评论、发PR更新

4. **优先验证最短链。**
   先证明一条最短路径：
   `不可信内容 -> 进入上下文 -> 触发工具或外传`

5. **最后评估实战性。**
   重点看：
   - 是否贴近默认部署
   - 是否依赖特定版本或少见配置
   - 是否需要高交互
   - 是否能稳定复现
   - 是否容易隐藏在正常工作流里

# 重点测试主题

### 1. 网页提示词注入

- 网页正文、隐藏文本、注释、评论区、PDF、跨标签页内容会不会被一起读入
- “总结当前页面”“帮我处理网页任务”这类常见动作会不会直接触发
- 浏览器工具是否能在已登录状态下跨站读敏感信息

### 2. 截图 / OCR 注入

- 截图中的小字、遮挡文本、提示层、图片文字会不会被模型当成指令
- Agent是否会基于截图内容继续执行点击、输入、导航
- 是否支持对截图进行多轮解析，导致恶意指令持续留在流程里

### 3. CI / PR / Issue 注入

- PR描述、Issue内容、提交信息、代码注释是否会直接进入Agent提示
- Agent在CI中能否调用Shell、代码修改、评论、提交、发布、拉取密钥
- 流水线里是否存在长期令牌、发布权限或云访问凭证

### 4. 消息通道与链接预览外传

- Agent回复中的链接是否会被Slack、Telegram等自动预览
- 链接预览请求是否会把模型拼接进去的数据直接带到攻击者域名
- 是否能做到“模型只要回复，预览就自动出网”

### 5. 批准绕过与沙箱逃逸

- 命令校验是否只看首命令名，不看子表达式、替换表达式或嵌套调用
- 沙箱边界是否能被下载脚本、子进程、外部解释器绕开
- 人工批准是否会因为展示内容被弱化而失效

# 实战性判断建议

可按这五项给每条方法打分：

- 默认可达性
- 所需交互量
- 对环境的依赖程度
- 触发后的权限高度
- 隐蔽性

经验上，下面这些情况通常更接近高实战性：

- 默认功能，不需要用户做非常规配置
- 入口来自正常工作流，如总结网页、跑CI、读Issue、回消息
- 触发后能直接接触令牌、登录态、仓库写权限或文件系统
- 用户界面很难完整展示真实动作与真实载荷

# 与其他skill的衔接

- 和`AI-IDE攻击面侦察`搭配：先确认网页、截图、CI、消息入口是否真实存在
- 和`提示词注入攻击链测试`搭配：验证外部内容能否稳定改变Agent行为
- 和`AI-IDE数据外传测试`搭配：验证浏览器、预览、回帖、HTTP请求是否能带出数据
- 和`终端过滤绕过测试`搭配：验证CI或浏览器子流程里的命令能否绕过限制
- 和`Agent工作区信任与审批测试`搭配：验证批准提示、信任模型、展示内容是否足够可靠

# 输出表述建议

最终建议写清楚：

- **入口**：恶意内容从哪里进入
- **触发动作**：用户最少做了什么
- **执行点**：浏览器、CI、消息通道、命令或其他工具
- **权限条件**：登录态、令牌、仓库权限、网络权限
- **外传或执行结果**：读到了什么、发到了哪里、改了什么
- **实战性评估**：高 / 中 / 低，以及原因
