# 常用命令过滤实现模式

这里总结AI IDE常见的命令过滤实现，以及每种实现容易出现的问题。

# 模式1：字符串前缀匹配**做法：**

只检查命令串是不是以允许命令开头。

**典型弱点：**
- 只看第一段
- 不看参数
- 不看后续拼接

# 模式2：只校验命令名**做法：**

把命令拆出第一个词，只判断它是否在allowlist里。

**典型弱点：**
- 危险参数可直接溜进去
- `find -exec`、`tar --to-command`这类很危险

# 模式3：审批显示与实际执行分离**做法：**

前端展示一个安全版命令，后端实际再拼接或包装。

**典型弱点：**
- 用户批的是A，实际跑的是B

# 模式4：不理解Shell语法**做法：**

把整行当普通字符串处理。

**典型弱点：**
- `$(...)`
- 反引号
- 换行
- 管道
- 分号

# 模式5：平台差异处理不一致**做法：**

Linux、macOS下测试过，Windows下逻辑却不同。

**典型弱点：**
- PowerShell的参数和展开方式完全不同
- `cmd /c`包装常引入新的边界错位
