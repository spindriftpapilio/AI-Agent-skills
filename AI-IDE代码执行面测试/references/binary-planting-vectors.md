# 各类IDE的二进制植入向量

目标：判断IDE或Agent会不会优先执行工作区中的同名程序。

# 常用优先目标

|二进制名|常用触发来源|打开项目时是否可能自动触发|备注|
|---|---|---|---|
|`git`|仓库状态、diff、分支信息|经常会|很多IDE会在打开项目时跑`git status`|
|`python` / `python3`|Python扩展、lint、venv|有时会|与扩展激活、解释器探测有关|
|`node` / `npm`|JS、TS工具链、任务|有时会|常与workspace task配合|
|`bash` / `sh`|task、脚本、终端包装器|视产品而定|留意包装层|

# 重点判断项

1. PATH搜索顺序是否把工作区目录放前面
2. 执行点是否来自扩展、Agent、任务系统还是核心IDE
3. Windows、Linux、macOS的行为是否一致
4. 同名脚本与同名可执行文件哪个优先

# 常用测试方式

- 在工作区放一个同名假的`git`或`python`
- 让它只输出一个可识别标记，不做破坏性动作
- 打开项目或触发相关功能，观察是否被调用
