# AI IDE审计用Semgrep规则模板

适合快速扫描开源代码库，优先抓常见危险模式。

# 规则方向1：字符串形式的命令执行重点找：

- `exec(<string>)`
- `spawn(..., {shell: true})`
- 由变量拼接出来的命令串

# 规则方向2：工作区配置驱动的危险调用重点找：

- 从`settings`、`config`、`workspace`读取后
- 直接用于命令、路径、URL、工具定义

# 规则方向3：弱审批实现重点找：

- 只校验名称不校验内容
- 审批结果长期缓存
- 修改后不重新验证

# 实施说明

- 先把规则写宽，再根据噪声收窄
- 和运行时证据对照，不要只靠静态命中下结论
