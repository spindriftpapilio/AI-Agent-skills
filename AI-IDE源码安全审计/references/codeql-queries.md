# AI IDE审计用CodeQL查询模板

以下是适合AI IDE场景的查询思路，重点看工作区输入如何流到高风险sink。

# 查询1：配置文件到命令执行**目标：**

- 找到从已知配置路径读取内容
- 最终流入`exec`、`spawn`、等价执行函数

**关注源（source）：**
- `settings.json`
- `mcp.json`
- rules / instructions文件

**关注汇（sink）：**
- 进程创建
- Shell执行
- 解释器路径设置

# 查询2：工作区文本到模型上下文再到工具调用**目标：**

- 看README、注释、规则文件是否能影响工具选择或命令参数

# 查询3：审批结果到后续复用**目标：**

- 找批准一次后被缓存的实现
- 看缓存key是路径、名字、哈希还是别的

# 使用说明

- 不必拘泥于固定规则名
- 先用广泛一点的查询圈定区域，再人工缩小
